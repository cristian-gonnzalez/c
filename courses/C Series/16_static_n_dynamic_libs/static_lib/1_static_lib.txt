Static libraries:

    .o files are static libraries. We can statically link together a series of .o files together to build a program. 
    .o files are compiled for specific architectures, so my .o file is unlikely to run on M1 Apple silicon for example.


- Compilation step by step:

   This is our project structure:

   $ tree
   .                                                                                                                                                                                                                             
   ├── include
   │   └── cmath.h
   └── src
       ├── cmath.c
       └── main.c

   2 directories, 3 files

1. Generate object files

   $ gcc -c ./src/main.c 
   ./src/main.c:13:10: fatal error: cmath.h: No such file or directory
      13 | #include "cmath.h"
         |          ^~~~~~~~~
   compilation terminated.
   $ gcc -c ./src/main.c -I./include
   $ ls
   include  main.o  src
            ~~~~~~
   $ gcc -c ./src/cmath.c -I./include
   $ ls
   cmath.o  include  main.o  src
   ~~~~~~~

Note:
   object files (.o) are compiled code. They contain symbol (names of variables and functions) of our compiled code.
   There several types of object files (static, shared executables) to be awared of.

2. Compile objects files

   $ gcc main.o cmath.o -o prog

3. Run the prog

   $ ./prog 
   add(4, 7) = 11
